# ABLATION 1 — Behavioral signals OFF
# Method: _behavioral_multiplier() patched to return 0.0 for all candidates.
# Effect: behavioral_multiplier feature = 0.0; feature weight still 0.10.
#         open_to_work, recruiter_response_rate, recency, interview_completion,
#         profile_completeness all zeroed out — 10% of weighted signal removed.
#
# NDCG@10: 0.3365   NDCG@50: 0.4340   MAP: 0.3080   P@10: 0.2000
#
# Delta vs baseline: NDCG@10 -0.4564 (-57.6%)
#
# Interpretation:
#   Dramatic drop shows behavioral signals are load-bearing.
#   Without open-to-work / recency signals, the LambdaMART model re-ranks
#   candidates whose pure domain + production scores are high but who are
#   passive / unreachable, bubbling them up at the expense of genuinely
#   active candidates that the golden set favours.
#
#   New rank #1 after behavioral OFF: CAND_0041611 (Staff MLE @ Locobuzz)
#   vs baseline rank #1: CAND_0018499 (Senior MLE @ Zomato, open-to-work).
#
#   Conclusion: behavioral_multiplier is the single most impactful feature
#   for discriminating between equally-qualified candidates. Do NOT ablate
#   or reduce its weight further.
#

# ABLATION 2 — Domain gate OFF (title_disqualified penalty disabled)
# Method: _title_disqualified() patched to return 0.0 for all candidates.
#         Candidates with disqualifying titles (HR Manager, Civil Engineer, etc.)
#         are no longer hard-gated to score 0.01.
#
# NDCG@10: 0.7929   NDCG@50: 0.6871   MAP: 0.5069   P@10: 0.4000
#
# Delta vs baseline: 0.0000 (identical)
#
# Top-10 with gate OFF: identical to baseline top-10
#
# Interpretation:
#   Zero NDCG change means none of the top-100 candidates are disqualified
#   by the gate — all 100 ranked candidates hold valid ML engineering titles.
#   The gate does its job silently: it prevents non-engineers from ever
#   entering the scoring competition. With the full 100k corpus, non-engineers
#   are already filtered by the dense + BM25 retrieval stage (they don't
#   match the JD embedding or keywords), so the gate acts as a safety net.
#
#   On the 50-candidate sample (Day 3), the gate DID cause visible rank changes
#   (removed HR Manager, Mechanical Engineer from top-10). On the full corpus
#   retrieval is strong enough that the gate is never triggered in the top-100.
#
#   Conclusion: gate is a defence-in-depth measure. Harmless when retrieval is
#   clean; critical backstop if retrieval quality degrades (e.g., weaker model,
#   distribution shift, adversarial keyword stuffing in profiles).
